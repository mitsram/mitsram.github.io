1. Overview & Context

1.1 What is GE ADMS?

GE ADMS (Advanced Distribution Management System) is a comprehensive real-time utility management platform used by electrical distribution operators. It integrates multiple operational capabilities:

  • SCADA (Supervisory Control and Data Acquisition) — Real-time monitoring and control of field devices
  • DMS (Distribution Management System) — Power flow analysis, fault management, network reconfiguration
  • OMS (Outage Management System) — Outage detection, crew dispatch, and restoration tracking
  • Network Modelling — Electrical connectivity models, schematic and geographic views
  • Switching Management — Safety-critical switching order creation and execution

The application presents a rich Windows desktop client with map views, one-line diagrams, tabular data grids, and complex modal workflows. This makes it an ideal (and challenging) candidate for desktop UI automation.

1.3 Key Automation Challenges in GE ADMS

  • Mixed UI Technologies: WPF panels mixed with legacy Win32 controls, DevExpress grids, and custom-rendered map views
  • Dynamic Element Trees: SCADA views load elements dynamically based on network model state
  • Custom-Rendered Controls: Map and schematic views use GDI+/DirectX rendering with no native UIA tree
  • Modal Workflow Chains: Switching operations involve multi-step modal dialogs with validation
  • Session State Dependencies: Tests depend on SCADA simulation state, database configurations, and user permissions
  • Long Load Times: Network model loading can take 30–120 seconds depending on model size
  • Third-Party Controls: DevExpress, Telerik, and Infragistics controls may require pattern-level interaction

2.3 UIA2 vs UIA3: Choosing the Right Provider

Feature UIA2 (Managed) UIA3 (COM-based) Recommendation for ADMS
WPF Controls Good Excellent Use UIA3 for WPF panels
Win32 / MFC Good Good Both work; UIA2 can be more stable for legacy
Custom Controls Pattern-based fallback Better pattern support UIA3 preferred
Performance Slightly faster (no COM marshalling) Slightly slower but more complete UIA3 in most cases

Recommendation: Start with UIA3 for GE ADMS automation. Fall back to UIA2 only for specific legacy dialogs where UIA3 has issues with element discovery.

4.2 Element Identification Strategies

Prioritize element identification in this order for maximum reliability:

  1. AutomationId — Most reliable, developer-assigned, rarely changes
  2. Name + ControlType — Good for labeled controls (buttons, menu items)
  3. ClassName + index — Use when AutomationId is absent
  4. XPath-like tree navigation — Fragile; use as last resort
  5. Coordinate-based — For custom-rendered elements only
// Strategy 1: AutomationId (Preferred)
var btnSwitch = window.FindFirstDescendant(cf =>
    cf.ByAutomationId("btnExecuteSwitch"));

// Strategy 2: Name + ControlType
var menuItem = window.FindFirstDescendant(cf =>
    cf.ByName("Network Operations").And(cf.ByControlType(ControlType.MenuItem)));

// Strategy 3: ClassName + Additional Condition
var grid = window.FindFirstDescendant(cf =>
    cf.ByClassName("DXGridControl").And(cf.ByName("Outage List")));

// Strategy 4: Chained navigation
var panel = window.FindFirstDescendant(cf => cf.ByAutomationId("MainPanel"));
var tab = panel.FindFirstChild(cf => cf.ByControlType(ControlType.Tab));
var tabItem = tab.FindAllChildren()
    .FirstOrDefault(c => c.Name == "Alarms");

AdmsApplication.cs — Application Lifecycle Manager

using FlaUI.Core;
using FlaUI.Core.AutomationElements;
using FlaUI.UIA3;
using System.Diagnostics;

namespace AdmsAutomation.Framework.Core;

public class AdmsApplication : IDisposable
{
    private Application? _application;
    private UIA3Automation? _automation;
    private Window? _mainWindow;

    public Window MainWindow => _mainWindow
        ?? throw new InvalidOperationException("ADMS not launched.");

    public UIA3Automation Automation => _automation
        ?? throw new InvalidOperationException("Automation not initialized.");

    /// <summary>
    /// Launch ADMS from the executable path.
    /// </summary>
    public void Launch(string executablePath, string arguments = "")
    {
        _automation = new UIA3Automation();

        var processStartInfo = new ProcessStartInfo(executablePath)
        {
            Arguments = arguments,
            WorkingDirectory = Path.GetDirectoryName(executablePath)
        };

        _application = Application.Launch(processStartInfo);

        // ADMS has a long startup; wait for the main window
        _mainWindow = WaitHelper.WaitUntil(() =>
        {
            try
            {
                return _application.GetMainWindow(_automation);
            }
            catch { return null; }
        },
        timeout: TimeSpan.FromMinutes(3),
        retryInterval: TimeSpan.FromSeconds(2));

        if (_mainWindow == null)
            throw new TimeoutException("ADMS main window did not appear within timeout.");
    }

    /// <summary>
    /// Attach to an already-running ADMS process.
    /// </summary>
    public void Attach(string processName = "ADMS.Client")
    {
        _automation = new UIA3Automation();
        var process = Process.GetProcessesByName(processName).FirstOrDefault()
            ?? throw new InvalidOperationException(
                $"No running process found with name '{processName}'.");

        _application = Application.Attach(process);
        _mainWindow = _application.GetMainWindow(_automation);
    }

    /// <summary>
    /// Attach by window title pattern (useful for multi-instance).
    /// </summary>
    public void AttachByTitle(string titleContains)
    {
        _automation = new UIA3Automation();
        var process = Process.GetProcesses()
            .FirstOrDefault(p => p.MainWindowTitle.Contains(
                titleContains, StringComparison.OrdinalIgnoreCase))
            ?? throw new InvalidOperationException(
                $"No window found containing '{titleContains}'.");

        _application = Application.Attach(process);
        _mainWindow = _application.GetMainWindow(_automation);
    }

    public void Dispose()
    {
        _automation?.Dispose();
        // Do NOT close ADMS by default; let teardown decide
    }

    public void CloseGracefully()
    {
        _mainWindow?.Close();
        _application?.WaitWhileMainHandleIsMissing(TimeSpan.FromSeconds(10));
        _application?.Close();
    }
}

WaitHelper.cs — Synchronization Utility

using FlaUI.Core.AutomationElements;
using FlaUI.Core.Conditions;

namespace AdmsAutomation.Framework.Core;

public static class WaitHelper
{
    /// <summary>
    /// Generic poll-based wait that returns a value.
    /// </summary>
    public static T? WaitUntil<T>(
        Func<T?> condition,
        TimeSpan timeout,
        TimeSpan? retryInterval = null) where T : class
    {
        var interval = retryInterval ?? TimeSpan.FromMilliseconds(500);
        var deadline = DateTime.UtcNow + timeout;

        while (DateTime.UtcNow < deadline)
        {
            var result = condition();
            if (result != null)
                return result;

            Thread.Sleep(interval);
        }

        return null;
    }

    /// <summary>
    /// Wait for a boolean condition to be true.
    /// </summary>
    public static bool WaitFor(
        Func<bool> condition,
        TimeSpan timeout,
        TimeSpan? retryInterval = null)
    {
        var interval = retryInterval ?? TimeSpan.FromMilliseconds(500);
        var deadline = DateTime.UtcNow + timeout;

        while (DateTime.UtcNow < deadline)
        {
            if (condition())
                return true;

            Thread.Sleep(interval);
        }

        return false;
    }

    /// <summary>
    /// Wait for a descendant element with retry.
    /// </summary>
    public static AutomationElement? WaitForElement(
        AutomationElement parent,
        Func<ConditionFactory, ConditionBase> conditionFunc,
        TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(30);

        return WaitUntil(() =>
        {
            try
            {
                return parent.FindFirstDescendant(conditionFunc);
            }
            catch
            {
                return null;
            }
        }, maxWait);
    }

    /// <summary>
    /// Wait for an element to become enabled.
    /// </summary>
    public static bool WaitForEnabled(
        AutomationElement element,
        TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(15);
        return WaitFor(() => element.IsEnabled, maxWait);
    }
}

BasePage.cs — Page Object Base

using FlaUI.Core;
using FlaUI.Core.AutomationElements;
using FlaUI.Core.Conditions;

namespace AdmsAutomation.Framework.Pages;

public abstract class BasePage
{
    protected readonly Window Window;
    protected readonly AutomationBase Automation;

    protected BasePage(Window window, AutomationBase automation)
    {
        Window = window;
        Automation = automation;
    }

    protected ConditionFactory CF => new ConditionFactory(
        new UIA3PropertyLibrary());

    /// <summary>
    /// Find a descendant element with built-in wait.
    /// </summary>
    protected AutomationElement FindElement(
        Func<ConditionFactory, ConditionBase> condition,
        TimeSpan? timeout = null)
    {
        var element = WaitHelper.WaitForElement(Window, condition, timeout);
        return element
            ?? throw new ElementNotFoundException(
                "Element not found within timeout.");
    }

    /// <summary>
    /// Safely check if an element exists without throwing.
    /// </summary>
    protected bool ElementExists(
        Func<ConditionFactory, ConditionBase> condition)
    {
        try
        {
            return Window.FindFirstDescendant(condition) != null;
        }
        catch { return false; }
    }

    /// <summary>
    /// Abstract validation that the page is loaded and ready.
    /// </summary>
    public abstract bool IsLoaded();
}

public class ElementNotFoundException : Exception
{
    public ElementNotFoundException(string message) : base(message) { }
}

using FlaUI.Core;
using FlaUI.Core.AutomationElements;

namespace AdmsAutomation.Framework.Pages;

public class LoginPage : BasePage
{
    public LoginPage(Window window, AutomationBase automation)
        : base(window, automation) { }

    // --- Element Properties ---

    private AutomationElement UsernameField =>
        FindElement(cf => cf.ByAutomationId("txtUsername"));

    private AutomationElement PasswordField =>
        FindElement(cf => cf.ByAutomationId("txtPassword"));

    private AutomationElement ServerDropdown =>
        FindElement(cf => cf.ByAutomationId("cmbServer"));

    private AutomationElement LoginButton =>
        FindElement(cf => cf.ByAutomationId("btnLogin"));

    private AutomationElement ErrorLabel =>
        FindElement(cf => cf.ByAutomationId("lblError"));

    // --- Actions ---

    public MainDashboard LoginAs(string username, string password,
                                  string? server = null)
    {
        EnterText(UsernameField, username);
        EnterText(PasswordField, password);

        if (server != null)
            SelectServer(server);

        LoginButton.Click();

        // Wait for main dashboard to appear
        WaitHelper.WaitFor(
            () => !Window.FindFirstDescendant(
                cf => cf.ByAutomationId("txtUsername"))?.IsOffscreen != true,
            TimeSpan.FromSeconds(60));

        return new MainDashboard(Window, Automation);
    }

    public string GetErrorMessage()
    {
        return ErrorLabel.AsLabel().Text;
    }

    public override bool IsLoaded()
    {
        return ElementExists(cf => cf.ByAutomationId("txtUsername")) &&
               ElementExists(cf => cf.ByAutomationId("btnLogin"));
    }

    // --- Private Helpers ---

    private void EnterText(AutomationElement element, string text)
    {
        var textBox = element.AsTextBox();
        textBox.Text = string.Empty;  // Clear existing
        textBox.Enter(text);
    }

    private void SelectServer(string serverName)
    {
        var combo = ServerDropdown.AsComboBox();
        combo.Expand();
        var item = combo.Items.FirstOrDefault(
            i => i.Text.Contains(serverName, StringComparison.OrdinalIgnoreCase));
        item?.Select();
    }
}

public class NetworkTreePanel : BasePage
{
    public NetworkTreePanel(Window window, AutomationBase automation)
        : base(window, automation) { }

    private AutomationElement TreeView =>
        FindElement(cf => cf.ByAutomationId("tvwNetwork"));

    /// <summary>
    /// Navigate a hierarchical path in the network tree.
    /// E.g., "Region-A" > "Substation-01" > "Feeder-03"
    /// </summary>
    public AutomationElement SelectEquipment(params string[] path)
    {
        var tree = TreeView.AsTree();
        AutomationElement current = tree;

        foreach (var nodeName in path)
        {
            // Expand current node
            var treeItem = current.FindFirstDescendant(
                cf => cf.ByName(nodeName)
                        .And(cf.ByControlType(ControlType.TreeItem)));

            if (treeItem == null)
                throw new ElementNotFoundException(
                    $"Tree node '{nodeName}' not found.");

            var expandCollapse = treeItem.Patterns.ExpandCollapse.Pattern;
            if (expandCollapse != null)
                expandCollapse.Expand();

            Thread.Sleep(500); // Wait for children to load
            current = treeItem;
        }

        current.Click();  // Select the final node
        return current;
    }

    /// <summary>
    /// Search for equipment by name using the filter box.
    /// </summary>
    public void SearchEquipment(string searchText)
    {
        var searchBox = FindElement(
            cf => cf.ByAutomationId("txtTreeSearch"));
        searchBox.AsTextBox().Enter(searchText);

        // Wait for filter to apply
        Thread.Sleep(1000);
    }

public class DevExpressGrid
{
    private readonly AutomationElement _gridElement;

    public DevExpressGrid(AutomationElement gridElement)
    {
        _gridElement = gridElement;
    }

    /// <summary>
    /// Get all rows from the grid.
    /// </summary>
    public AutomationElement[] GetRows()
    {
        return _gridElement.FindAllDescendants(
            cf => cf.ByControlType(ControlType.DataItem));
    }

    /// <summary>
    /// Get cell value by row index and column name.
    /// </summary>
    public string GetCellValue(int rowIndex, string columnName)
    {
        var rows = GetRows();
        if (rowIndex >= rows.Length)
            throw new IndexOutOfRangeException(
                $"Row index {rowIndex} exceeds row count {rows.Length}.");

        var row = rows[rowIndex];
        var gridItemPattern = row.Patterns.GridItem.PatternOrDefault;

        // Try using the Grid pattern
        var gridPattern = _gridElement.Patterns.Grid.PatternOrDefault;
        if (gridPattern != null)
        {
            // Find column index by header name
            var headers = _gridElement.FindAllDescendants(
                cf => cf.ByControlType(ControlType.HeaderItem));
            var colIndex = headers
                .Select((h, i) => new { h.Name, Index = i })
                .FirstOrDefault(x => x.Name == columnName)?.Index ?? -1;

            if (colIndex >= 0)
            {
                var cell = gridPattern.GetItem(rowIndex, colIndex);
                return cell.Name ?? cell.Patterns.Value.PatternOrDefault?.Value
                       ?? string.Empty;
            }
        }

        // Fallback: search within row for named element
        var cellElement = row.FindFirstDescendant(
            cf => cf.ByName(columnName));
        return cellElement?.Name ?? string.Empty;
    }

    /// <summary>
    /// Find a row by matching a column value.
    /// </summary>
    public AutomationElement? FindRow(string columnName, string value)
    {
        var rows = GetRows();
        for (int i = 0; i < rows.Length; i++)
        {
            var cellValue = GetCellValue(i, columnName);
            if (cellValue.Contains(value, StringComparison.OrdinalIgnoreCase))
                return rows[i];
        }
        return null;
    }

    /// <summary>
    /// Click a row to select it.
    /// </summary>
    public void SelectRow(int rowIndex)
    {
        var rows = GetRows();
        rows[rowIndex].Click();
    }

    /// <summary>
    /// Double-click a row (often opens a detail dialog in ADMS).
    /// </summary>
    public void DoubleClickRow(int rowIndex)
    {
        var rows = GetRows();
        rows[rowIndex].DoubleClick();
    }

    /// <summary>
    /// Get total row count.
    /// </summary>
    public int RowCount => GetRows().Length;

    /// <summary>
    /// Sort by clicking a column header.
    /// </summary>
    public void SortByColumn(string columnName)
    {
        var header = _gridElement.FindFirstDescendant(
            cf => cf.ByName(columnName)
                    .And(cf.ByControlType(ControlType.HeaderItem)));
        header?.Click();
    }

    /// <summary>
    /// Scroll the grid to load virtualized rows.
    /// </summary>
    public void ScrollToBottom()
    {
        var scrollPattern = _gridElement.Patterns.Scroll.PatternOrDefault;
        if (scrollPattern != null && scrollPattern.VerticallyScrollable)
        {
            while (scrollPattern.VerticalScrollPercent < 100)
            {
                scrollPattern.Scroll(ScrollAmount.NoAmount,
                                      ScrollAmount.LargeIncrement);
                Thread.Sleep(200);
            }
        }
    }
}

public class DialogHandler
{
    private readonly Window _mainWindow;

    public DialogHandler(Window mainWindow)
    {
        _mainWindow = mainWindow;
    }

    /// <summary>
    /// Wait for and return the active modal dialog.
    /// </summary>
    public Window? WaitForModal(TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(10);
        return WaitHelper.WaitUntil(
            () => _mainWindow.ModalWindows.FirstOrDefault(),
            maxWait);
    }

    /// <summary>
    /// Dismiss a dialog by clicking a button.
    /// </summary>
    public void DismissDialog(string buttonName)
    {
        var dialog = WaitForModal();
        if (dialog == null) return;

        var button = dialog.FindFirstDescendant(
            cf => cf.ByName(buttonName)
                    .And(cf.ByControlType(ControlType.Button)));
        button?.Click();
    }

    /// <summary>
    /// Handle unexpected pop-up dialogs (errors, warnings).
    /// </summary>
    public bool HandleUnexpectedDialogs()
    {
        var modals = _mainWindow.ModalWindows;
        foreach (var modal in modals)
        {
            var title = modal.Title;
            if (title.Contains("Error") || title.Contains("Warning"))
            {
                // Log the unexpected dialog
                var message = modal.FindFirstDescendant(
                    cf => cf.ByControlType(ControlType.Text))?.Name;

                // Try to close it
                var okButton = modal.FindFirstDescendant(
                    cf => cf.ByName("OK"));
                var closeButton = modal.FindFirstDescendant(
                    cf => cf.ByName("Close"));

                (okButton ?? closeButton)?.Click();
                return true;  // An unexpected dialog was handled
            }
        }
        return false;
    }

    /// <summary>
    /// Get dialog message text (for assertion).
    /// </summary>
    public string GetDialogMessage()
    {
        var dialog = _mainWindow.ModalWindows.FirstOrDefault();
        if (dialog == null)
            return string.Empty;

        var textElements = dialog.FindAllDescendants(
            cf => cf.ByControlType(ControlType.Text));

        return string.Join(" ",
            textElements.Select(t => t.Name).Where(n => !string.IsNullOrEmpty(n)));
    }
}

7.1 Wait & Synchronization Strategies

GE ADMS has varying load times depending on network model size and server response. A robust wait strategy is critical.

public static class AdmsWaitStrategies
{
    /// <summary>
    /// Wait for the ADMS "busy" spinner to disappear.
    /// Many ADMS operations show a loading overlay.
    /// </summary>
    public static void WaitForBusyIndicatorToDisappear(
        Window window, TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(120);

        WaitHelper.WaitFor(() =>
        {
            var spinner = window.FindFirstDescendant(
                cf => cf.ByAutomationId("busyIndicator"));
            return spinner == null || spinner.IsOffscreen;
        }, maxWait);
    }

    /// <summary>
    /// Wait for a SCADA value to update.
    /// </summary>
    public static bool WaitForScadaValue(
        AutomationElement element,
        string expectedValue,
        TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(30);

        return WaitHelper.WaitFor(() =>
        {
            var currentValue = element.Patterns.Value.PatternOrDefault?.Value
                               ?? element.Name;
            return currentValue.Contains(expectedValue,
                StringComparison.OrdinalIgnoreCase);
        }, maxWait);
    }

    /// <summary>
    /// Wait for the network model to finish loading.
    /// </summary>
    public static void WaitForModelLoad(
        Window window, TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromMinutes(3);

        // Wait for status bar to show "Model Loaded" or "Ready"
        WaitHelper.WaitFor(() =>
        {
            var statusBar = window.FindFirstDescendant(
                cf => cf.ByAutomationId("statusBar"));
            var text = statusBar?.FindAllChildren()
                .Select(c => c.Name)
                .Aggregate("", (a, b) => a + " " + b) ?? "";

            return text.Contains("Ready", StringComparison.OrdinalIgnoreCase)
                || text.Contains("Model Loaded",
                    StringComparison.OrdinalIgnoreCase);
        }, maxWait);
    }
}

/// <summary>
/// Wrapper for DevExpress GridControl which may not expose
/// standard DataGrid patterns.
/// </summary>
public static class DevExpressHelper
{
    /// <summary>
    /// DevExpress grids sometimes use custom row presentation.
    /// Fall back to raw descendant search when Grid pattern is absent.
    /// </summary>
    public static AutomationElement[] GetGridRows(
        AutomationElement gridElement)
    {
        // Try standard DataItem pattern first
        var rows = gridElement.FindAllDescendants(
            cf => cf.ByControlType(ControlType.DataItem));

        if (rows.Length > 0) return rows;

        // Fallback: DevExpress may use Custom control type
        rows = gridElement.FindAllDescendants(
            cf => cf.ByControlType(ControlType.Custom)
                    .And(cf.ByClassName("DataRowView")));

        if (rows.Length > 0) return rows;

        // Fallback: try ListItem
        return gridElement.FindAllDescendants(
            cf => cf.ByControlType(ControlType.ListItem));
    }

    /// <summary>
    /// For DevExpress ComboBox that doesn't use standard ComboBox pattern.
    /// </summary>
    public static void SelectDevExpressComboItem(
        AutomationElement comboBox, string itemText)
    {
        comboBox.Click();
        Thread.Sleep(300);

        // DevExpress drops a popup window
        var popup = comboBox.FindFirstDescendant(
            cf => cf.ByControlType(ControlType.List));

        if (popup == null)
        {
            // Try finding the popup as a top-level window
            popup = comboBox.Automation.GetDesktop()
                .FindFirstDescendant(
                    cf => cf.ByControlType(ControlType.List)
                            .And(cf.ByClassName("PopupListBox")));
        }

        var item = popup?.FindFirstDescendant(
            cf => cf.ByName(itemText));
        item?.Click();
    }
}

/// <summary>

{
    /// <summary>
    /// Find a child window by title.
    /// </summary>
    public static Window? FindChildWindow(
        Application app, AutomationBase automation, string titleContains)
    {
        return app.GetAllTopLevelWindows(automation)
            .FirstOrDefault(w => w.Title.Contains(
                titleContains, StringComparison.OrdinalIgnoreCase));
    }

    /// <summary>
    /// Wait for a new window to appear (e.g., undocked panel).
    /// </summary>
    public static Window? WaitForNewWindow(
        Application app, AutomationBase automation,
        int previousWindowCount, TimeSpan? timeout = null)
    {
        var maxWait = timeout ?? TimeSpan.FromSeconds(15);

        return WaitHelper.WaitUntil(() =>
        {
            var windows = app.GetAllTopLevelWindows(automation);
            if (windows.Length > previousWindowCount)
                return windows.Last();
            return null;
        }, maxWait);
    }

    /// <summary>
    /// Bring a specific window to the foreground.
    /// </summary>
    public static void BringToFront(Window window)
    {
        window.SetForeground();
        Thread.Sleep(300);
    }

8.3 Retry & Resilience Patterns

/// <summary>
/// Retry attribute for flaky ADMS interactions.
/// </summary>
[Test]
[Retry(3)]
[Category("Flaky")]
public void ScadaValue_ShouldUpdateAfterControl()
{
    // This test may be timing-sensitive due to SCADA communication delays
}

/// <summary>
/// Custom retry logic for element interaction.
/// </summary>
public static class RetryHelper
{
    public static T RetryOnException<T>(
        Func<T> action,
        int maxRetries = 3,
        TimeSpan? delayBetweenRetries = null)
    {
        var delay = delayBetweenRetries ?? TimeSpan.FromSeconds(1);

        for (int i = 0; i < maxRetries; i++)
        {
            try
            {
                return action();
            }
            catch when (i < maxRetries - 1)
            {
                Thread.Sleep(delay);
            }
        }

        return action(); // Final attempt; let exception propagate
    }
}

9. CI/CD Integration

9.1 Agent Configuration

Critical Requirement: FlaUI (and all UI Automation tools) requires an interactive desktop session. Tests will fail on headless agents or services running as SYSTEM.

Agent Requirement Configuration
Desktop Session Agent must run as a user with auto-logon; use RDP or console session
Screen Resolution Minimum 1920×1080; match the development/inspection resolution
DPI Scaling Set to 100% to avoid coordinate mismatches
Power Settings Disable sleep, screen saver, and lock screen
ADMS Client Pre-installed and configured to connect to the test SCADA server

9.3 Parallel Execution Considerations

UI automation tests cannot run in parallel on the same desktop session. Each FlaUI test interacts with the live desktop, so concurrent tests will conflict. Strategies to mitigate:

  • Sequential execution on a single agent (safest)
  • Multiple VMs with one agent per VM for parallel suites
  • Test categories to split suites across agents (Smoke on VM1, Functional on VM2)
  • RDP multi-session with separate user sessions per test runner
<!-- NUnit configuration to enforce sequential execution -->
<!-- Add to .runsettings file -->
<RunSettings>
  <NUnit>
    <NumberOfTestWorkers>1</NumberOfTestWorkers>
  </NUnit>
</RunSettings>

10. Troubleshooting & Common Pitfalls

Problem Cause Solution
Elements not found Element tree not loaded; wrong automation type (UIA2 vs UIA3) Use FlaUInspect to verify the tree. Try switching between UIA2 and UIA3. Add waits.
Click has no effect Element is covered by an overlay, or coordinates are off-screen Use element.SetForeground() before clicking. Verify element is enabled and not obscured.
Tests pass locally, fail on CI No interactive desktop session; different screen resolution/DPI Ensure agent has auto-logon, 1920×1080 resolution, 100% DPI scaling.
DevExpress grid returns no rows DevExpress uses virtualization; only visible rows are in UIA tree Scroll the grid before searching. Use the ScrollToBottom() helper.
Modal dialog not detected Dialog may be a child of a different window, or a popup rather than modal Check Window.ModalWindows, try app.GetAllTopLevelWindows(), inspect the desktop.
Map elements not interactable Custom-rendered GDI+/DirectX control with no UIA tree Use coordinate-based clicks. Use ADMS search/locate features. Consider API-level shortcuts.
Stale element references UI has been refreshed; cached element is invalid Always re-find elements before interaction. Use property-based lookups (lazy properties) in page objects.
Slow test execution Excessive Thread.Sleep calls or deep element tree searches Use intelligent waits. Narrow search scope with FindFirstChild instead of FindFirstDescendant where possible.
COMException during automation UIA3 COM interop failure; sometimes caused by ADMS window transitions Wrap interaction in RetryHelper.RetryOnException(). Update FlaUI to the latest version.
Text not entered in fields Control doesn’t support ValuePattern; may need keyboard simulation Use Keyboard.Type() as fallback. Click the field first, then type.
